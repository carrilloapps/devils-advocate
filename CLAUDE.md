# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repository Is

A collection of **AI agent skills** — pure Markdown instruction files (no executable code) distributed through the [skills.sh](https://skills.sh) ecosystem. Skills are installed into target projects via `npx skills add carrilloapps/skills@<skill-name>`. The only executable artifact is `scripts/validate.sh`.

Two skills are published:
- **devils-advocate** (`skills/devils-advocate/`) — Mandatory adversarial analysis gate. Runs FIRST before any action; blocks execution until user approves (`✅ Proceed`).
- **sar-cybersecurity** (`skills/sar-cybersecurity/`) — Automated Security Assessment Report (SAR) generator with bilingual EN/ES output and 20+ compliance standards.

## Mandatory: Devil's Advocate Gate

**Before proposing or implementing ANY change to this repository**, load and apply `skills/devils-advocate/SKILL.md`. All actions require adversarial analysis and explicit user approval before proceeding. This is enforced by `AGENTS.md`.

## Commands

### Validation (run before every commit)
```bash
bash scripts/validate.sh
```
Runs 49 automated checks: version consistency, fence balance, Gate block completeness, `continue` wording, index completeness, stale text detection, version stamps in examples, framework file resolution, token budget, and more. All checks must pass before pushing.

There is no build step, no test suite, no linter beyond `validate.sh`. This is a pure Markdown project.

## Architecture

### Skill Structure

Each skill follows the same pattern:
- `SKILL.md` — Core instruction file, always loaded in full by agents. Must stay under ~8,000 tokens (~32,000 chars).
- `frameworks/` — Modular files loaded on demand. Split into **protocol files** (free to load) and **domain frameworks** (max 2 per analysis).
- `examples/` — Reference outputs showing correct protocol application.
- `metadata.json` — Version, author, keywords.
- `README.md` — Public documentation.

### Progressive Context Loading

Skills use a budget system to avoid context window saturation:
- **Protocol files** are free (always loadable).
- **Domain frameworks** are capped at 2 per analysis.
- **Examples** are loaded on demand only when needed.

This architecture is why `SKILL.md` is kept small — it's the always-loaded entry point that references heavier files.

### Version Cascade

When bumping a skill version, the version must be updated in **all** of these locations:
1. `SKILL.md` frontmatter (`version:`)
2. `README.md` version badge
3. `metadata.json` `"version"` field
4. All `examples/*.md` files (`**Skill version**: X.Y.Z`)
5. Root `README.md` skill catalog badge
6. Root `CHANGELOG.md` new section

`validate.sh` catches mismatches between these (checks 1, 8, 11, 12).

### Cross-Skill Interaction

Both skills share the same repo but are independently installable. Devils-advocate acts as the orchestration gate that runs before sar-cybersecurity (or any other action). The planned `postmortem-writing` skill will close the feedback loop: DA (prevent) → SAR (assess) → Postmortem (learn).

## Conventions

- **Commits**: Conventional Commits (`feat:`, `fix:`, `docs:`)
- **Language**: `en_US` for code identifiers and documentation
- **Branch**: `main` only
- **No AI credit**: Never add `Co-Authored-By` with any AI/IDE name, "Generated by" markers, or tool watermarks. All credit belongs to the human user.
- **Git operations**: Never commit without explicit user authorization, even with full session permissions.

## Security Audit Compliance

Every skill must pass three security scanners on [skills.sh/audits](https://skills.sh/audits):
- **Gen Agent Trust Hub**: SAFE (no remote code execution, no external downloads)
- **Socket**: PASS 4/4 (no malicious behavior, no credential exposure, no obfuscation, no excessive autonomy)
- **Snyk**: LOW RISK (untrusted input boundaries, scoped web search, report-only output)

Six mandatory safeguards must appear in every `SKILL.md`: untrusted input boundary, no arbitrary code execution, bounded autonomy, web search scoping, example code boundaries, report-only output. See `.ai-context.md` for the full checklist.
